
**Описание решения кейса**
I.	Выбранный кейс: Программа конвертации валют с использованием API
II.	Цель: разработать программу на Python для конвертации валют с получением актуальных курсов через открытое API. Использовать ИИ для улучшения кода.
Задачи:
1.	Создать функцию конвертации между популярными валютами (USD, EUR, RUB)
2.	Добавить простой консольный интерфейс для ввода суммы и выбора валют.
3.	Использовать ИИ-инструмент для генерации кода обработки исключений.
4.	Реализовать логирование операций конвертации.
5.	Написать тесты для основных функций.
III.	Обзор использованных технологий и инструментов, включая ИИ-инструменты:
Для написания кода использована версия Python 3.13. 
Код реализован в среде разработки (IDE) Visual Studio Code, наиболее удобных из доступных бесплатно для разработки приложений
Для разработки приложения использован пакет PyQt6, отличающаяся повышенной гибкостью в сочетании с простотой в создании интерфейса.
Для улучшения кода использована доступная бесплатно версия ChatGPT 5.5

**Аналитическая часть**
Для выполнения задания в первую очередь была написана функция для конвертации валют (convert()), которая принимает информацию об имеющейся и желаемой валютах, сумму в имеющейся валюте, дату конвертации и размер коэффициента, на который требуется умножить, что осуществить конвертацию. Поскольку предполагается, что пользователь не ограничен в том, как заполнять соответствующее поле, была включена обработка исключений на случай ввода нечисловых значений. В ситуации, когда все-таки введено число, по умолчанию переменным коэффициента и даты присваиваются значения None. Если пользователь на вход функция принимает одинаковые значения для имеющейся и запрашиваемой валют, коэффициент принимает значение 1, а дата не обновляется, поскольку в этой ситуации значения на входе и выходе всегда будут одинаковыми. В противном случае коэффициент и дата обновляются. Если все-таки функция получает для коэффициента значение None, пользователь получает сообщение об отсутствии сети. В остальных ситуациях сумма округляется до сотых специальным методом и конвертируется в формат с плавающей точкой. Переменная результата рассчитывается путем умножения коэффициента на сумму и последующим специальным округлением до сотых. Если дата по какой-то причине не получена, то выводится только результат конвертации, в противном случае, отдельно еще и дата.
Для получения коэффициента была написана отдельная функция (get_coef()), также принимающая информацию об имеющейся и желаемой валютах, а также сохраненную на предыдущих итерациях кода информацию из кэша. Данные о коэффициенте получается при помощи функции get пакета reports из открытого API Frankfurter с максимальном временем ожидания сервера 5 секунд. В адрес страницы вносятся изменения в зависимости от имеющейся и желаемой валют. Из получаемого формата json извлекается значение коэффициента. Далее объявляется переменная даты, которая фиксирует дату и время на момент извлечения коэффициента. Затем информация о коэффициенте и дате вносится в виде вложенного словаря и сохраняется кэш для вероятного использования в будущем. В случае неудачи при попытке подключения к API, вызывается исключение, которое в случае наличия информации в кэше извлекает оттуда последние актуальные коэффициент и дату, в противном случае присваивает соответствующим переменным None.
Для кэширования данных э коэффициентах и датах их загрузки были созданы файл json для хранения информации, функция для загрузки информации из кэша (load_cache()) и функция сохранения информации в кэш (save_cache()). Первая возвращает ее содержимое в случае существования, в противном случае – пустой словарь. Вторая сохраняет информацию.
Поскольку мной было принято решение разработать дружелюбное для пользователя приложение, не требующее обращения к командной строке, я обратился к пакету PyQt, предоставляющее возможность сравнительно небольшими усилиями получить работающее приложение с современным дизайном. В частности, я создал класс, наследующий от класса Qwiget. При помощи функций пакета был создан заголовок, установлены размеры окна. На случай расширения окна все содержимое остается в центре. Устанавливаются два раскрывающихся списка, поле для открытого ввода и кнопка для конвертации. Также кнопки для обратной конвертации для удобства пользователя. В конце два имплицитных поля для вывода: для результата конвертации большим зеленым шрифтом и шрифтом поменьше для даты. Далее все виджеты, собственно, добавляются в layout. Поскольку функции конвертации и получения коэффициента требуют информации, регулируемой пользователем через интерфейс, они были помещены в класс
Наконец, в классе создаю функцию для обратной конвертации, которая меняет местами имеющуюся и востребованную валюты, после чего вызывает функцию конвертации.
